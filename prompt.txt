Build an AI-powered Gmail Reply Agent with the following features:

1. Email Parsing & Context
- Read incoming Gmail messages using Gmail API.
- Extract:
  - Sender
  - Subject
  - Body
  - Thread ID
- Maintain conversation context using thread history.

2. Writing Style Learning
- Analyze user's previous sent emails.
- Learn:
  - Tone (formal/informal)
  - Greeting style
  - Closing style
  - Sentence structure
- Use this to mimic user's writing style in replies.

3. Thread Understanding
- Use Gmail threadId to fetch previous emails.
- Understand:
  - Conversation flow
  - Time gaps between replies
  - Context continuity
- Generate replies that are context-aware (not standalone).

4. Multi-LLM Support (via OpenRouter)
- Integrate with OpenRouter API.
- Support models:
  - OpenAI (GPT)
  - Anthropic (Claude)
  - Google (Gemini)
- Allow dynamic model selection.

5. Confidence Filtering
- Each generated reply should include a confidence score.
- If confidence >= 80%:
    → Auto-generate draft reply
- Else:
    → Mark as “Needs Review”

6. Special Case Handling
- Detect OTP / transactional emails.
- Do NOT generate replies for:
  - OTPs
  - Alerts
  - No-reply emails
- Instead classify them as “No Response Needed”.

7. Draft Generation
- Save generated replies as Gmail drafts.
- Do not auto-send emails.

8. Language Adaptation
- Match language of incoming email.
- If user typically replies in mixed language (e.g., English + Hindi), adapt accordingly.

9. System Architecture
- Backend: Node.js / Python
- Gmail API integration
- OpenRouter API for LLM calls
- Database to store:
  - Email embeddings
  - Style patterns
  - Thread summaries

10. Optional Enhancements
- Summarize long threads before generating replies
- Add “Reply Suggestions” (multiple options)
- UI dashboard to review drafts

Goal:
Create a smart assistant that reads emails, understands context and writing style, and generates high-quality reply drafts with confidence scoring.
