System Design Preparation Notes

1. Handle open-ended problem statements  
   - Clearly define assumptions  
   - Ask clarifying questions before jumping to solution  

2. High-Level Design (HLD)  
   - Identify core components (frontend, backend, services)  
   - Define system architecture (monolith / microservices)  
   - Draw component interaction flow  

3. APIs Design  
   - Define REST/GraphQL endpoints  
   - Specify request/response structure  
   - Include authentication & authorization  
   - Handle error responses properly  

4. Database Design  
   - Choose database type (SQL vs NoSQL)  
   - Design schema / collections  
   - Define relationships and indexing  
   - Consider scalability and performance  

5. Data Flow  
   - Explain how data moves across components  
   - Request → Processing → Storage → Response  
   - Include caching and queue mechanisms if needed  

6. Get Plan from AI (like ChatGPT)  
   - Ask for initial approach  
   - Compare with alternative approaches  
   - Evaluate trade-offs (scalability, cost, complexity)  

7. Optimization & Edge Cases  
   - Handle failures and retries  
   - Plan for scaling (horizontal/vertical)  
   - Consider security and rate limiting  

8. Alternative Solutions  
   - Always think of at least one different design  
   - Compare pros and cons
